# PicGallery 🖼️

PicGallery 是一套**为家庭和极客设计的高级局域网/NAS 多媒体画廊管理系统**。通过深邃现代的玻璃拟态 UI（Glassmorphism）和高度优化的引擎，PicGallery 能够以极高的加载效率呈现存储在服务器乃至 NAS 里的海量照片与视频回忆。

系统内置了热更新后台权限控制与双语支持，既能满足自己强大的数据统筹欲，也能给长辈、亲人提供极简傻瓜式的瀑布流沉浸浏览体验。

---

## ✨ 核心特性

- **极致瀑布流与深度穿透检索**：自动向深层穿透扫描并映射多重文件夹（DFS 递归），所有潜藏在不同年份与事件里的图纸均能被收纳汇总并以紧凑的瀑布流画廊平铺而出。
- **全格式双模引擎**：支持常见的图片矩阵，还集成了对主流视频的扫描以及封面的帧提取（依赖底层的 `ffmpeg` 驱动）。
- **完全国际化（i18n 引擎）**：系统拥有自行内建的反应字典上下文（Context），用户可点击右上角的地球仪一键瞬时在中文/English之间平滑全站切换！
- **热更新与动态后台设定**：所有系统架构级的变动（例如重定向挂载的新 NAS 照片源、修改超级管理员凭证等）都可以直接在网页端后台表单输入生效，免去了繁杂的配置文件重启流程。
- **媒体隐形屏蔽网**：您可以针对任意敏感或不想展示出的照片或视频一键加盖阻断层，所有的普通访问者将被隐瞒其存在，从而只展现你所期望展示的美好图库。
- **智能高斯缩略与毫秒调度**：后端使用强力图形引擎（`sharp`）高速裁切下发基于 WebP 的轻量缩略图并本地缓冲存盘，有效避免加载上万张全画幅原图导致的网络堵塞与浏览器卡死。
- **时序与资源聚合雷达**：前端支持实时的“全视频过滤”、“全照片过滤”以及**基于机器源文件指纹按年份自动生成的切片聚合下拉栏**，您可以迅速直达您人生任意一年的记忆片场。

## 🛠️ 服务器运行需求

PicGallery 为了保证图纸加载速度以及处理海量照片的稳定性，采用了纯 Node.js 双模架构打底。您可以将其直接部署在 Linux (推荐，如 Ubuntu、Debian)、Windows 以及 Mac 设备之上。

### 基础环境：
- **Node.js** v18 或更高版本
- **一台挂载了照片的任意设备**（可以是本地硬盘，也可以是挂接上的 NAS/SMB 网络共享驱动器）

### 视频抽帧组件要求（如果需要浏览视频）：
请确保您系统底层装有基础的 `ffmpeg`，并在终端命令行可运行：
- Ubuntu 体系：`sudo apt install ffmpeg`
- CentOS 体系：`sudo yum install ffmpeg`

## 🚀 部署指南

### 1. 拷贝与获取项目代码
首先将本仓库搬运到您计划运行的服务终端里：
```bash
git clone <本代码仓库地址> PicGallery
cd PicGallery
```

### 2. 双端构建流配置
本系统分为后端服务层与前端渲染层，需要分别挂接依赖。

**A. 安装并编译前端系统：**
```bash
cd frontend
npm install
npm run build
```
*(上述命令会将 React 工程完美抽真空打包到 `dist` 文件夹，后端将会自动托管挂载这批产品)*

**B. 挂接后端运转引擎：**
```bash
cd ../backend
npm install
```

### 3. 一键出厂唤醒
一切就绪后，保留在 `backend` 文件夹，您可以敲入以下命令启动整套系统：
```bash
node server.js
```
*(提示：在生产环境中极为推荐采用 PM2 进行后台常驻守护与开机自启：`pm2 start server.js --name picgallery`)*

### 4. 初始化登入 
1. 启动完成后，在同一局域网下的任意设备的浏览器访问 `http://<您的服务器IP>:3000`
2. 点击右上角的 **“🔒 管理登录”**
3. 默认验证密钥：
   - 账号：`admin`
   - 密码：`admin123`
4. 成功登录后，请务必立刻前往 **齿轮图标设置** 板块修改您的初始暗号，并将 **“NAS挂载网络完整根目录”** 链接上您想开放展现的真实图库地址！

*(注意：系统在接纳新目录后，会在后台默默对所有图纸进行缩压备份和重载。期间如果有黑图为正常现象，耐心等待一两分钟即可体验极速翻阅之旅)*

    └── package.json          # Vite 工程清单

## 🪧 其它运维备忘

- **隐私隔离保护：** 未登录用户仅仅只能查看且被强行套用隐身网（被屏蔽图纸对非管理者全层级物理隔绝，且所有接口强行验证 Admin 签名拦截扫描）。
- **极速热加载：** 如果用 NAS 等外部渠道向库里塞了新文件，不需要重启服务，系统下一次刷新或者上传文件时将会顺便把未识别的缺口给一并读取生成。
- **安全存放分离：** 无论下属文件夹层叠嵌套多深，在 PicGallery 上点击直接强行上传的图纸会被**默认甩在总根目录**上，不会因您的临时上传搞乱您的有序文件树形归类结构。
